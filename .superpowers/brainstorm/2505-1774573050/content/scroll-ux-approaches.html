Sticky Progress Header

Persistent header showing current position (Q12/82), progress bar, and context. Scroll indicator shows where you are in the list. Current question highlighted with accent border.

Pros

  • Always know your position
  • Progress visible at all times
  • Minimal UI changes needed

Cons

  • Takes vertical space on mobile
  • Still a long scroll

Question Number Jump Bar + Virtual Scroll

Sticky numbered grid at top - tap any number to jump. Color-coded: green=correct, red=incorrect, blue=current, gray=unattempted. Virtualized list renders only visible items for 60fps scrolling even with 200+ questions.

Pros

  • Jump to any question instantly
  • See attempt status at a glance
  • Butter-smooth with 200+ items

Cons

  • Needs react-window/virtuoso
  • Jump bar takes space

One-at-a-Time Swipe Mode (Kindle-style)

Full-screen question view with swipe left/right navigation. Bottom mini-map shows all questions as colored dots. No scrolling - just swipe through questions like a Kindle book. List mode still available as toggle.

Pros

  • Zero scrolling fatigue
  • Focused, distraction-free
  • Great mobile UX (swipe)

Cons

  • Can't scan multiple questions
  • Slower to browse/skim
  • Biggest code change

My Recommendation: Hybrid B + A

Combine the jump bar with status dots (B) for quick navigation with the sticky progress header (A) for context awareness. Keep the scrollable list (not one-at-a-time) since teachers need to scan/edit many questions quickly. Add virtual scrolling only if performance degrades with 100+ items. The Kindle swipe mode (C) is great as an optional toggle for students in practice/study mode.
