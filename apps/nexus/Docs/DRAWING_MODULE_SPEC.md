## 1. Executive Summary

This document specifies the Drawing Practice Module for the Nexus classroom platform. The module addresses NATA drawing preparation through five integrated sub-modules:

1. **Question Bank** — Browse & practice with recalled NATA drawing questions
2. **Object Library** — Standalone reference image library organized by object family (3 difficulty levels)
3. **Foundation Checklist** — Track foundation skill completion per student
4. **Art Gallery** — Social wall of student submissions (Instagram-style)
5. **Tutor Review & Sketch-Over** — Admin panel for evaluating student drawings with canvas overlay

## 2. NATA 2025 Drawing Question Analysis

### 2.1 Question Pattern

Every NATA 2025 drawing paper follows a consistent **3-question pattern** (90 minutes total, 80 marks):

| Category | Type | Medium | Marks | Key Skills |
|----------|------|--------|-------|------------|
| **A: 2D Composition** | Shape-based OR Theme-based design (logos, posters, patterns, signage, product surfaces) | Colored. Color constraints always specified. | ~25 | Composition, color theory, design thinking, lettering |
| **B: 3D Object Composition** | Still-life of everyday objects with shade & shadow | Black & white pencil only | ~25 | Form drawing, light/shadow, texture, proportion, depth |
| **C: 3D Model/Sculpture (Kit)** | Arrange ~12 given physical cutout shapes on cardboard to depict a design principle | Physical cutouts + glue on cardboard | ~30 | Spatial thinking, design principles, 3D visualization |

### 2.2 Category A: 2D Composition Sub-types

| Sub-type | Examples | Frequency |
|----------|----------|-----------|
| Geometric shape composition | Parallelograms, pentagons, semicircles, rectangles, trapezoids with color constraints | ~40% |
| Logo/badge design | Circular logo for charity marathon, corporate wellness challenge water bottle | ~15% |
| Poster/signage design | Film festival poster, photography exhibition, park signage, airport check-in signage | ~15% |
| Product surface design | Tote bag (bird motifs), baby blanket (vehicle shapes), carpenter apron (tool shapes), piano cover (music notes), T-shirt (adventure theme) | ~20% |
| Wall mural / art book cover | Peepal leaf mural, kindergarten art book poster | ~10% |

### 2.3 Category B: 3D Object Families

Students who can draw these from memory will be prepared for most questions:

| Object Family | Specific Objects Asked | Occurrence |
|---------------|----------------------|------------|
| **Fruits** | Orange, apple, pomegranate, papaya, mango, pineapple, muskmelon, grapes, kiwi, strawberry, melon | Very High |
| **Kitchen/Dining** | Woven basket, glass tumblers, ceramic bowls, napkin rings, forks, salad plates, place mat, spoons (tea/table/serving), rice cup, earthen pots, laundry basket, cloth | High |
| **Travel/Outdoor** | Travel bag, rolling suitcase, hat, walking stick, water bottle, binoculars, neck pillow, backpack, toiletries bag | High |
| **Lighting** | Street lamp, garden lamp, oil lamp, candle, desk clamp, pedestal lamp, flood lamp | Medium |
| **Sports** | Football, cricket ball, tennis ball, racket, tennis net, goal net, skateboard, cycle wheel, helmet, cleats, cricket cap | Medium |
| **Stationery** | Pencil, eraser, sharpener, stapler, paint tubes, paint bottles, brush | Medium |
| **Toiletries/Accessories** | Comb, lipstick, shampoo bottle, talcum powder, vanity box, sunglasses, headphone, umbrella | Medium |
| **Misc** | Books (stacked), pyramids, spices in basket, broom, bucket, dolls, toy cars, soft toys, flower vase | Low-Medium |

### 2.4 Category C: Kit Sculpture Themes

| Theme | Frequency |
|-------|-----------|
| Spiral Tower / Spiraling Tower | Very High (most common) |
| Off-Centric Tower | High |
| Emphasis | High |
| Dynamism / Dynamic Structure | Medium |
| Contrast | Medium |
| Symmetry | Medium |
| Rhythm | Low-Medium |
| Repetition | Low-Medium |
| Movement / Flow | Low-Medium |
| Flight / Wave | Low |
| Fort / Cave | Low |
| Imbalance / Twisted Tower | Low |

### 2.5 Color Constraints Pattern

| Constraint Type | Examples |
|----------------|----------|
| Primary colors only | Red, blue, yellow |
| Secondary colors only | Green, orange, purple |
| Analogous colors | Adjacent on color wheel (e.g., blue-green-teal) |
| Warm colors / warm hues | Red, orange, yellow families |
| Specific hue families | Shades of blue, shades of red and grey |
| Dark colors | For showing depth |
| Max N colors (2, 3, or 4) | Most common constraint format |

## 3. Drawing Training Curriculum

### 3.1 Track 1: Foundation Skills (Weeks 1–3)

**Week 1: Hand & Line Foundation**
- Line practice: straight lines (horizontal, vertical, diagonal, varying pressure)
- Curved lines: arcs, S-curves, spirals, freehand circles
- Basic 2D shapes freehand: circle, square, triangle, rectangle, pentagon, hexagon, parallelogram, trapezoid, semicircle
- Shape variation: different sizes, orientations, overlapping, intersecting

**Week 2: Color Theory & 3D Form Basics**
- Color wheel: primary, secondary, tertiary
- Color relationships: analogous, complementary, warm/cool, monochromatic
- Coloring techniques: flat fill, gradient, layering with pencils
- 3D basic forms: sphere, cube, cylinder, cone, pyramid — light direction, core shadow, cast shadow, reflected light, highlight
- Hatching, cross-hatching, blending, stippling

**Week 3: Composition & Texture**
- Composition principles: rule of thirds, golden ratio, balance, visual weight
- Overlap, intersection, negative space, depth through size variation
- Texture rendering: smooth (glass, metal), rough (fabric, wood, woven basket), organic (fruit skin, leaves)
- Scale, proportion, human anthropometry basics
- How to draw common forms: trees, flowers, human figures (basic), lettering
- Design principles vocabulary: symmetry, emphasis, dynamism, contrast, rhythm, repetition, movement, flow, balance

### 3.2 Track 2: Object Library Practice (Weeks 3–6)

AI-generated reference images (via ChatGPT/Gemini) at 3 difficulty levels:

- **Level 1 (Beginner):** Clean outline drawing with basic proportions labeled
- **Level 2 (Intermediate):** Outline + basic shading with light direction indicated
- **Level 3 (Advanced):** Fully rendered with texture, detailed shading, shadow, and highlights

| Week | Object Families | 2D Practice | Kit Concepts |
|------|----------------|-------------|--------------|
| Week 3 | Fruits (spherical: orange, apple, mango) + Basic forms (pyramid, cube) | Geometric shape compositions with color constraints | Symmetry, Balance |
| Week 4 | Fruits (complex: pineapple, grapes, pomegranate) + Kitchen items | Logo/badge design, product surface design | Emphasis, Contrast |
| Week 5 | Travel items (bags, hats, bottles) + Lighting (lamps, candles) | Poster design, signage | Dynamism, Movement, Rhythm |
| Week 6 | Sports items + Stationery + Toiletries/Accessories | Mural, pattern design, complete 2D variety | Spiral Tower, Off-Centric Tower, Flow, Repetition |

### 3.3 Track 3: Exam Simulation (Weeks 5–8)

Full timed mock sessions replicating NATA exam conditions. Each session = 3 questions (one from each category).

- Week 5–6: Semi-timed practice (relaxed timing, focus on completion quality)
- Week 7: Full timed mocks with self-assessment
- Week 8: Full timed mocks with tutor evaluation + final weak-area drilling

### 3.4 AI Image Generation Prompt Template

```
"Generate a [LEVEL] reference drawing of [OBJECTS] arranged in a still-life
composition. [LEVEL-SPECIFIC INSTRUCTIONS]. The image should be suitable as
a practice reference for architecture entrance exam students. White background,
clean presentation."
```

- **Level 1 suffix:** "Show only clean outlines with no shading. Label basic proportions. Simple, clear line drawing style."
- **Level 2 suffix:** "Show outlines with basic light and shadow. Indicate light source direction with an arrow. Medium detail, visible construction lines."
- **Level 3 suffix:** "Fully rendered photorealistic pencil drawing with detailed textures, accurate light source from top-left, cast shadows, reflected light, and surface detail. Professional quality."

### 4.2 Sub-Module 1: Drawing Question Bank

#### 4.2.1 Data Model — `drawing_questions`

```sql
CREATE TABLE drawing_questions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  year INTEGER NOT NULL,
  session_date DATE,
  source_student TEXT,
  category TEXT NOT NULL CHECK (category IN ('2d_composition', '3d_composition', 'kit_sculpture')),
  sub_type TEXT NOT NULL,
  -- For 2D: geometric_shapes | logo | poster | signage | product_surface | mural
  -- For 3D: still_life
  -- For Kit: the theme keyword (spiral_tower, emphasis, dynamism, etc.)
  question_text TEXT NOT NULL,
  objects TEXT[] DEFAULT '{}',
  color_constraint TEXT,
  -- e.g. 'primary_colors_only', 'max_3_colors', 'analogous_colors', 'black_and_white'
  design_principle TEXT,
  -- For kit: symmetry, emphasis, dynamism, contrast, rhythm, etc.
  difficulty_tag TEXT NOT NULL DEFAULT 'medium' CHECK (difficulty_tag IN ('easy', 'medium', 'hard')),
  reference_images JSONB DEFAULT '[]',
  -- Array of {level: 1|2|3, url: string, alt_text: string}
  solution_images JSONB,
  -- Tutor/AI generated solution images
  tags TEXT[] DEFAULT '{}',
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Indexes
CREATE INDEX idx_dq_category ON drawing_questions(category);
CREATE INDEX idx_dq_year ON drawing_questions(year DESC);
CREATE INDEX idx_dq_tags ON drawing_questions USING GIN(tags);
CREATE INDEX idx_dq_difficulty ON drawing_questions(difficulty_tag);
CREATE INDEX idx_dq_sub_type ON drawing_questions(sub_type);
```

#### 4.2.2 Student-Facing UI

**Browse View (List/Grid):**
- Filter by: Year, Category (2D/3D/Kit), Sub-type, Object family, Design principle, Difficulty
- Each question card shows: question text, category badge (color-coded), difficulty badge, object tags, thumbnail of Level 2 reference image
- Sort by: Year (desc), Difficulty, Recently added

**Question Detail View:**
- Full question text prominently displayed
- Reference Images section with **3-level toggle** (Beginner / Intermediate / Advanced)
- "Practice This" button → opens submission flow
- "View Solutions" toggle → shows tutor/AI-generated solution images (hidden by default to encourage attempt first)
- Related questions carousel (same object family or same sub-type)

#### 4.2.3 Practice & Submission Flow

```
Student taps "Practice This"
    ↓
Student draws on paper
    ↓
Student photographs work
    ↓
Student uploads photo (camera capture or gallery pick — PWA optimized for mobile)
    ↓
Optional: Student adds self-reflection note
    ↓
Submission saved → status: 'submitted'
    ↓
Appears in Tutor Review Queue (admin panel)
    ↓
Tutor reviews with sketch-over + rating + feedback
    ↓
Student sees toggle: "My Drawing" | "Tutor Feedback"
    ↓
Optionally published to Art Gallery
```

#### 4.2.4 Data Model — `drawing_submissions`

```sql
CREATE TABLE drawing_submissions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID NOT NULL REFERENCES profiles(id),
  question_id UUID REFERENCES drawing_questions(id),
  -- null for free-form homework or free practice
  homework_id UUID REFERENCES drawing_homework(id),
  -- null if self-initiated practice
  source_type TEXT NOT NULL CHECK (source_type IN ('question_bank', 'homework', 'free_practice')),
  original_image_url TEXT NOT NULL,
  -- Student's uploaded photo
  reviewed_image_url TEXT,
  -- Tutor's sketch-over composite version
  self_note TEXT,
  -- Student's self-reflection ("I struggled with the shadow on the basket")
  ai_feedback JSONB,
  -- AI-generated feedback (when enabled)
  -- {score: 'B', feedback: ['Good composition', 'Shadow direction inconsistent', ...]}
  tutor_rating INTEGER CHECK (tutor_rating BETWEEN 1 AND 5),
  tutor_feedback TEXT,
  status TEXT NOT NULL DEFAULT 'submitted'
    CHECK (status IN ('submitted', 'under_review', 'reviewed', 'published')),
  is_gallery_published BOOLEAN DEFAULT false,
  submitted_at TIMESTAMPTZ DEFAULT now(),
  reviewed_at TIMESTAMPTZ,

  CONSTRAINT valid_source CHECK (
    (source_type = 'question_bank' AND question_id IS NOT NULL) OR
    (source_type = 'homework' AND homework_id IS NOT NULL) OR
    (source_type = 'free_practice')
  )
);

-- Indexes
CREATE INDEX idx_ds_student ON drawing_submissions(student_id);
CREATE INDEX idx_ds_status ON drawing_submissions(status);
CREATE INDEX idx_ds_gallery ON drawing_submissions(is_gallery_published) WHERE is_gallery_published = true;
CREATE INDEX idx_ds_submitted ON drawing_submissions(submitted_at DESC);
CREATE INDEX idx_ds_question ON drawing_submissions(question_id);
```

### 4.3 Sub-Module 2: Tutor Review & Sketch-Over

#### 4.3.1 Review Queue (Admin Panel)

**Location:** `admin.neramclasses.com → Drawing Reviews`

- Queue shows all submissions with status = `submitted`, sorted by submission time (oldest first)
- Filter by: student, question category, date range, batch
- Bulk actions: mark multiple as reviewed, assign ratings
- Dashboard widget: pending review count, average review turnaround time

#### 4.3.2 Sketch-Over Tool

This is the flagship feature. When tutor opens a submission for review:

1. Student's original image loads as the **base canvas layer**
2. Tutor gets a **transparent overlay layer** for sketching corrections
3. **Drawing tools:** freehand pen (red default, configurable color), line tool, circle tool, eraser, undo/redo
4. **Pen thickness:** thin (for detail notes), medium (for corrections), thick (for emphasis)
5. **Text annotation tool:** tap anywhere to add a text label (e.g., "shadow too light here", "good texture!")
6. Tutor adds **rating (1–5 stars)** and optional **text feedback**
7. **"Save Review"** → composites the overlay onto the image and saves as `reviewed_image_url`
8. **"Publish to Gallery"** toggle → optionally pushes to Art Gallery

**Technical Implementation:**

```
┌─────────────────────────────────┐
│  Canvas Container               │
│  ┌───────────────────────────┐  │
│  │  Layer 0: Student Image   │  │ ← <img> or canvas drawImage
│  │  Layer 1: Tutor Overlay   │  │ ← HTML5 Canvas (transparent)
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │  Toolbar                   │  │
│  │  [Pen] [Line] [Circle]    │  │
│  │  [Text] [Eraser] [Undo]  │  │
│  │  [Color] [Thickness]      │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │  Rating: ★★★★☆            │  │
│  │  Feedback: [_________]    │  │
│  │  [Save Review] [Publish]  │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

- Use **HTML5 Canvas API** (or Fabric.js for richer features) for the sketch overlay
- Store overlay as **separate PNG layer** + composite on save
- Student sees **toggle:** "My Drawing" | "Tutor Feedback" to switch between original and reviewed versions
- Fluent UI v9 components for the review interface

### 4.4 Sub-Module 3: Object Library

A standalone browseable reference library of objects organized by family, each with 3-level reference images. Exists independently from the question bank.

```sql
CREATE TABLE drawing_objects (

  reference_images JSONB NOT NULL DEFAULT '[]',
  -- [{level: 1, url: string}, {level: 2, url: string}, {level: 3, url: string}]

  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_do_family ON drawing_objects(family);
CREATE INDEX idx_do_tags ON drawing_objects USING GIN(tags);
```

### 4.5 Sub-Module 4: Foundation Checklist

A structured skills progression tracker mapped to the Week 1–3 curriculum. Each student has their own checklist state.

#### 4.5.1 Data Model

```sql
-- Master checklist items (seeded once)
CREATE TABLE drawing_checklist_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category TEXT NOT NULL,
  -- 'hand_practice', 'shapes_2d', 'color_theory', 'forms_3d',
  -- 'shading_techniques', 'textures', 'composition', 'design_principles', 'special_topics'
  skill_name TEXT NOT NULL,
  sort_order INTEGER NOT NULL,
  description TEXT
);

-- Per-student progress
CREATE TABLE drawing_checklist_progress (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID NOT NULL REFERENCES profiles(id),
  checklist_item_id UUID NOT NULL REFERENCES drawing_checklist_items(id),
  status TEXT NOT NULL DEFAULT 'not_started'
    CHECK (status IN ('not_started', 'in_progress', 'completed')),
  student_marked_at TIMESTAMPTZ,
  tutor_verified BOOLEAN DEFAULT false,
  tutor_verified_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ DEFAULT now(),

  UNIQUE(student_id, checklist_item_id)
);

CREATE INDEX idx_dcp_student ON drawing_checklist_progress(student_id);
```

#### 4.5.2 Checklist Items

| Category | Skill Items |
|----------|-------------|
| **Hand Practice** | Straight lines (H/V/D), Curved lines & arcs, Freehand circles (various sizes), Spiral drawing, Pressure control (light to dark) |
| **2D Shapes** | Circle, Square, Rectangle, Triangle, Pentagon, Hexagon, Parallelogram, Trapezoid, Semicircle, Ellipse |
| **Color Theory** | Primary colors identification, Secondary colors mixing, Analogous colors, Complementary colors, Warm vs Cool, Monochromatic shading |
| **3D Basic Forms** | Sphere (with shading), Cube (with perspective), Cylinder, Cone, Pyramid, Combined forms |
| **Shading Techniques** | Hatching, Cross-hatching, Blending/smudging, Stippling, Light direction understanding, Cast shadow, Reflected light |
| **Textures** | Smooth surface (glass/metal), Rough surface (wood/stone), Organic (fruit skin), Woven/fabric, Transparent/translucent |
| **Composition** | Rule of thirds, Overlapping objects, Size variation for depth, Negative space, Visual balance |
| **Design Principles** | Symmetry, Emphasis, Contrast, Rhythm, Repetition, Dynamism, Movement, Flow, Balance |
| **Special Topics** | Human figure (basic proportions), Hand drawing, Tree & nature, Perspective (1-point, 2-point), Lettering, Anthropometry basics |

#### 4.5.3 Checklist Mechanics

- Each item has 3 states: **Not Started** (grey) → **In Progress** (yellow) → **Completed** (green)
- Student self-marks progress, tutor can override/verify
- Overall completion percentage shown as progress bar on student dashboard
- Tutor dashboard shows **class-wide checklist heatmap** (which skills are lagging)
- Suggested next practice based on incomplete checklist items

```sql
  submission_id UUID NOT NULL REFERENCES drawing_submissions(id) ON DELETE CASCADE,
  student_id UUID NOT NULL REFERENCES profiles(id),
  comment_text TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_dgr_submission ON drawing_gallery_reactions(submission_id);
CREATE INDEX idx_dgc_submission ON drawing_gallery_comments(submission_id);
```

### 4.7 AI Feedback Layer (Future Phase)

#### 4.7.1 Phase 1: Free Basic Feedback

Using Claude Vision API or GPT-4V, provide instant basic feedback:

- **Composition check:** Is the drawing centered? Is the frame well-utilized?
- **Proportion feedback:** Are objects roughly proportional to each other?
- **Shading check:** Is there a clear light direction? Are shadows present?
- **Completeness:** Does the drawing address all objects mentioned in the question?
- **Output:** 4–5 bullet points of constructive feedback + overall score (A/B/C/D)

### 5.4 RLS Policies

```sql
-- Students: read all questions, read own submissions, read gallery, create submissions/reactions/comments
ALTER TABLE drawing_questions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Questions readable by all authenticated" ON drawing_questions
  FOR SELECT TO authenticated USING (is_active = true);

ALTER TABLE drawing_submissions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Students read own submissions" ON drawing_submissions
  FOR SELECT TO authenticated USING (student_id = auth.uid());
CREATE POLICY "Students create submissions" ON drawing_submissions
  FOR INSERT TO authenticated WITH CHECK (student_id = auth.uid());

-- Gallery: published submissions readable by all
CREATE POLICY "Gallery readable by all" ON drawing_submissions
  FOR SELECT TO authenticated USING (is_gallery_published = true);

-- Tutors: read/write all submissions, manage questions/homework
-- (Use role-based check via profiles.role or custom claim)
CREATE POLICY "Tutors manage submissions" ON drawing_submissions
  FOR ALL TO authenticated USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('tutor', 'admin'))
  );
```
